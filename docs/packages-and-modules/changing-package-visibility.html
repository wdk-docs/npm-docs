更改包可见性¶

您可以从网站或命令行更改范围包的可见性。

您必须是拥有该程序包的用户帐户或组织的所有者才能更改程序包可见性。

For more information about package visibility, see npm包范围，访问级别和可见性.

注解

Adding a scope to an unscoped package will not make it private. If you want to make a unscoped package private, contact npm Support.

使公共包私有化¶

注解

Making a package private requires a paid user account or Org. To sign up for a paid user or Org, go to https://www.npmjs.com/settings/account-name/billing, replacing account-name with the name of your npm user account or Org.

If you want to restrict access and visibility for a public package you own, you can make the package private. When you make a package private, it will be removed from the website within a few minutes of the change.

网站¶

  1. 在 npm website 上，转到包页面。

  2. 在程序包页面上，单击“管理员”。

  3. 在“Package Access”下，选择“Package is Private？”

  4. 单击“更新包设置”。

命令行¶

要在命令行上创建公共包私有，请运行以下命令，将<package-name>替换为包的名称:

npm access restricted <package-name>

有关更多信息，请参阅 npm access 文档。

公开私人套餐¶

注解

当您将私有包公开时，所有npm用户都可以看到该包并可由其下载。

网站¶

  1. 在 npm website 上，转到包页面。

  2. 在包页面上，单击 Admin 。

  3. 在“Package Access”下，取消选择“Package is Private？”

  4. 单击 更新包设置 。

命令行¶

要在命令行上公开私有包，请运行以下命令，将 <package-name> 替换为包的名称:

npm access public <package-name>

有关更多信息，请参阅 npm access CLI文档。
