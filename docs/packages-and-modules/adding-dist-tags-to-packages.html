将dist-tags添加到包中¶

分发标签（dist-tags）是人类可读的标签，可用于组织和标记您发布的不同版本的软件包。 dist-tags补充 关于语义版本控制 。除了比语义版本编号更易于阅读之外，标签还允许发布者更有效地分发其包。

For more information, see the npm dist-tag.

注解

由于dist-tags与语义版本共享命名空间，因此请避免与现有版本号冲突的dist-tag。我们建议避免使用以数字或字母 “v” 开头的dist-tag。

使用dist-tag发布包¶

默认情况下，运行npm publish会使用最新的dist-tag标记您的包。要使用另一个dist-tag，请在发布时使用–tag标志。

  1. 在命令行上，导航到程序包的根目录。

cd /path/to/package

  1. 运行以下命令，将<tag>替换为要使用的标记:

npm publish --tag <tag>

示例

要在命令行上发布带有“beta”dist-tag的软件包，请在软件包的根目录中运行以下命令:

npm publish --tag beta

将dist标记添加到特定版本的包中¶

在命令行上，导航到程序包的根目录。

cd /path/to/package

运行以下命令，将<package_name>替换为程序包的名称，将<version>替换为程序包版本号，将<tag>替换为分发标记:

npm dist-tag add <package-name>@<version> [<tag>]

示例

要将 “stable” 标记添加到 “example-package” 软件包的1.4.0版本，您将运行以下命令:

npm dist-tag add example-package@1.4.0 stable
