npm-adduser¶

概要¶

adduser¶

Add a registry user account

npm adduser [--registry=url] [--scope=@orgname] [--always-auth] [--auth-type=legacy]

aliases: login, add-user

描述¶

Create or verify a user named <username> in the specified registry, and save the credentials to the .npmrc file. If no registry is specified, the default registry will be used (see npm-config).

用户名，密码和电子邮件将从提示中读入。

要重置密码，请转到https://www.npmjs.com/forgot

要更改您的电子邮件地址，请转到 https://www.npmjs.com/email-edit

You may use this command multiple times with the same user account to authorize on a new machine. When authenticating on a new machine, the username, password and email address must all match with your existing record.

npm login是adduser的别名，行为方式完全相同。

配置¶

注册¶

默认: https://registry.npmjs.org/

The base URL of the npm package registry. If scope is also specified, this registry will only be used for packages with that scope. scope defaults to the scope of the project directory you’re currently in, if any. See npm-scope.

范围¶

默认: none

If specified, the user and login credentials given will be associated with the specified scope. See npm-scope. You can use both at the same time, e.g.

npm adduser –registry=http://myregistry.example.com –scope=@myco This will set a registry for the given scope and login or create a user for that registry at the same time.

always-auth¶

默认: false

如果已指定，请保存配置，指示对给定注册表的所有请求都应包含授权信息。对私人注册表有用。可以与–registry和/或–scope一起使用，例如

npm adduser --registry=http://private-registry.example.com --always-auth

这将确保对该注册表（包括tarball）的所有请求都包含授权标头。与私有注册表一起使用时，此设置可能是必需的，其中元数据和包tar包存储在具有不同主机名的主机上。有关always-auth的更多详细信息，请参阅 npm-config 中的 always-auth 。特定于注册表的always-auth配置优先于任何全局配置。

auth-type¶

  • Default: ‘legacy’

  • Type: ‘legacy’, ‘sso’, ‘saml’, ‘oauth’

adduser/login 使用什么身份验证策略。除了传统的 npm 中的经典用户名/密码输入之外，一些npm注册表（例如，npmE）可能支持替代身份验证策略。
